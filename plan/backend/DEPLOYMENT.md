# Python后端部署方案

## 1. 概述

由于Cloudflare Workers不支持Python，本方案提供了多种替代部署方式，确保Python后端能够稳定运行并与前端正常通信。

## 2. 本地开发环境

### 2.1 安装Python

确保安装Python 3.10或更高版本：

```bash
# 检查Python版本
python --version
# 或
python3 --version
```

### 2.2 安装依赖

```bash
# 进入后端目录
cd backend

# 创建虚拟环境（可选但推荐）
python -m venv venv

# 激活虚拟环境
# macOS/Linux
source venv/bin/activate
# Windows
venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt
```

### 2.3 配置环境变量

创建`.env`文件并配置环境变量：

```bash
# 复制示例文件
cp .env.example .env

# 使用编辑器修改.env文件
# 添加你的OpenAI API密钥等配置
```

### 2.4 运行开发服务器

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

服务器将在`http://localhost:8000`运行，API文档可在`http://localhost:8000/docs`访问。

## 3. Docker容器化部署

### 3.1 安装Docker

确保安装Docker和Docker Compose：

```bash
# 检查Docker版本
docker --version

# 检查Docker Compose版本
docker-compose --version
```

### 3.2 构建Docker镜像

```bash
# 进入后端目录
cd backend

# 构建镜像
docker build -t endgame-backend .
```

### 3.3 运行Docker容器

```bash
# 运行单个容器
docker run -d -p 8000:8000 --env-file .env endgame-backend

# 或使用docker-compose（推荐）
# 在项目根目录创建docker-compose.yml
# 然后运行
docker-compose up -d
```

### 3.4 Docker Compose配置

在项目根目录创建`docker-compose.yml`：

```yaml
version: '3.8'

services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    environment:
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      - HOST=0.0.0.0
      - PORT=8000
      - DEBUG=False
    volumes:
      - ./backend:/app
    command: uvicorn main:app --host 0.0.0.0 --port 8000

  frontend:
    build: .
    ports:
      - "5173:5173"
    volumes:
      - .:/app
      - /app/node_modules
    command: npm run dev
    depends_on:
      - backend
```

## 4. 云平台部署

### 4.1 AWS部署

#### 4.1.1 使用ECS + Fargate

1. 创建ECR仓库存储Docker镜像
2. 构建并推送镜像到ECR
3. 创建ECS集群
4. 配置Task Definition
5. 创建Service并部署

#### 4.1.2 使用Elastic Beanstalk

1. 安装EB CLI
2. 初始化EB应用
3. 部署应用

```bash
# 初始化EB应用
cd backend
 eb init -p python-3.10 endgame-backend

# 部署应用
 eb create endgame-backend-env
```

### 4.2 GCP部署

#### 4.2.1 使用Cloud Run

1. 安装Google Cloud CLI
2. 构建并部署到Cloud Run

```bash
# 构建并部署
cd backend
gcloud run deploy endgame-backend \
  --source . \
  --platform managed \
  --region us-central1 \
  --allow-unauthenticated \
  --set-env-vars OPENAI_API_KEY=${OPENAI_API_KEY}
```

### 4.3 Azure部署

#### 4.3.1 使用App Service

1. 安装Azure CLI
2. 登录Azure
3. 创建资源组
4. 创建App Service计划
5. 部署应用

```bash
# 登录Azure
az login

# 创建资源组
az group create --name endgame-rg --location eastus

# 创建App Service计划
az appservice plan create --name endgame-plan --resource-group endgame-rg --sku B1 --is-linux

# 部署应用
cd backend
az webapp up --name endgame-backend --resource-group endgame-rg --plan endgame-plan --runtime "PYTHON|3.10"
```

### 4.4 Vercel/Netlify部署

虽然Vercel和Netlify主要用于前端部署，但也支持Python函数：

#### 4.4.1 使用Vercel Serverless Functions

1. 在项目根目录创建`api`文件夹
2. 创建Python函数文件
3. 配置Vercel项目

#### 4.4.2 使用Netlify Functions

1. 在项目根目录创建`netlify/functions`文件夹
2. 创建Python函数文件
3. 配置Netlify项目

注意：这些平台的函数有执行时间限制，可能不适合长时间运行的WebSocket连接。
